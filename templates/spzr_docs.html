⚠️ Важное замечание о работе СППР

Вердикт (качественный/брак) НЕ ЗАВИСИТ от текущего значения Δx!

Согласно методичке (стр. 35-38), Δx используется только для расчета градаций n, но итоговый вердикт определяется пороговым правилом P ≤ 0.5, которое является константой. В нашей реализации:

  • Базовый вердикт вычисляется один раз с фиксированным Δx = 1.0 и НЕ МЕНЯЕТСЯ
  • Текущий Δx влияет только на отображение градаций (n), log₂(n), Co, Go и текущее P
  • Количество качественных и бракованных товаров остается постоянным при любом Δx

Пример: При Δx=0.1 градации большие, при Δx=5 градации маленькие, но товар, признанный качественным при базовом расчете, остается качественным всегда.

1. Расчет градаций (стр. 35)

Формула из методички:

n = ⎧ 2, если значение характеристики в норме
    ⎨ (X - Xmax)/Δx + 1, если значение характеристики > max значения нормы
    ⎩ (Xmin - X)/Δx + 1, если значение характеристики < min значения нормы

Пояснение переменных:

  • X — реальное значение характеристики
  • Xmin — минимальное значение нормы
  • Xmax — максимальное значение нормы
  • Δx — шаг измерения (чувствительность системы)
  • n — количество градаций (всегда ≥ 2)

Реализация в коде с правильным округлением:

Важное исправление: используем math.ceil() вместо int() для правильного округления вверх

def calculate_gradations(x, xmin, xmax, dx):
    """
    Правильный расчет градаций по методичке
    """
    if xmin <= x <= xmax:
        return 2
    elif x > xmax:
        # Отклонение вверх
        diff = x - xmax
        n = math.ceil(diff / dx) + 1
        return max(2, min(n, 100))
    else:  # x < xmin
        # Отклонение вниз
        diff = xmin - x
        n = math.ceil(diff / dx) + 1
        return max(2, min(n, 100))

Пример расчета для разных Δx:

Для характеристики "Износостойкость" с нормой 7-10 и реальным значением 3.76:

Δx Разница (Xmin - X) (Xmin - X)/Δx ceil() n
0.1 3.24 32.4 33 34
1.0 3.24 3.24 4 5
5.0 3.24 0.648 1 2

Вывод: Чем меньше Δx, тем больше градаций, тем чувствительнее система к отклонениям.

2. Показатели Ch, Co, Go (стр. 37)

Ch

Сигнал нормы

Ch = N

где N — количество характеристик (всегда 8)

Co

Сигнал отклонения

Co = Σ log₂(nᵢ)

сумма по 8 характеристикам

Go

Отношение

Go = Co / Ch

показывает степень отклонения

Пример расчета для разных Δx (один и тот же товар):

Для товара с 4 отклонениями:

Δx Средние n Co Go P (текущее) Базовое P Вердикт
0.1 12.5 11.25 1.406 0.712 0.632 БРАК
1.0 3.8 8.25 1.031 0.521
5.0 2.1 8.00 1.000 0.500

Важно: Базовое P (при Δx=1.0) = 0.632 > 0.5, поэтому товар БРАК. Текущее P меняется, но вердикт остается!

3. Вероятность правильной классификации (стр. 38)

Формула из методички:

P = e-ln2 / Go²

где ln2 ≈ 0.693147

Анализ формулы:

  • Чем больше Go (сильнее отклонения), тем меньше P
  • Чем меньше P, тем выше уверенность в классификации
  • При Go = 1: P = e-0.693 = 0.5 (граничное значение)
  • При Go = 1.2: P = e-0.693/1.44 = e-0.481 = 0.618
  • При Go = 0.8: P = e-0.693/0.64 = e-1.083 = 0.339

4. Пороговое решающее правило (стр. 38)

Цитата из методички:

"За диапазон нормы принимается значение вероятности P от 0 до 0,5"

✅ Качественный товар

P ≤ 0.5

Система уверена в качестве продукции. Отклонения незначительны.

❌ Брак

P > 0.5

Система обнаружила критические отклонения. Товар не соответствует нормам.

🔍 Важное уточнение:

В нашей реализации базовое P вычисляется с Δx = 1.0 и именно оно определяет вердикт. Текущее P при других Δx отображается только для демонстрации чувствительности системы.

5. Влияние шага измерения Δx

Что такое Δx?

Δx (шаг измерения) — минимальная единица шкалы, определяющая чувствительность системы.

Важно: Δx стоит в знаменателе формулы градаций

n = ceil((x - xmax)/Δx) + 1 или n = ceil((xmin - x)/Δx) + 1

Сравнение для разных Δx (с фиксированным вердиктом):

Δx Средний Go Средний log₂(n) Качественные Брак % брака
0.1 1.722 1.89 11 37 77%
1.0 1.315 1.42 25 23 48%
5.0 1.042 1.08 36 12 25%

Вывод: Чем меньше Δx (0.1) → тем больше градаций → тем больше кажется брака (77%)
Чем больше Δx (5.0) → тем меньше градаций → тем меньше кажется брака (25%)
НО базовый вердикт (при Δx=1.0) остается неизменным!

6. Кнопка "Подобрать оптимальный Δx"

Принцип работы:

Кнопка автоматически перебирает различные значения Δx и находит оптимальное, при котором количество качественных и бракованных товаров максимально сбалансировано.

Алгоритм работы:

  1. Перебирает значения Δx: 0.1, 0.2, 0.5, 0.8, 1.0, 1.5, 2.0, 3.0, 5.0
  2. Для каждого Δx вычисляет процент качественных товаров
  3. Находит Δx, при котором процент качественных ближе всего к 50%
  4. Устанавливает это значение как оптимальное

Зачем это нужно:

Из методички (стр. 49-50):

  • "Обучение СППР... можно сделать вывод о том, правильно ли функционирует система"
  • "сдвинуть порог P" или "уменьшить шаг Δх"
  • Оптимальный Δx дает наиболее сбалансированную классификацию

Пример с текущими данными:

Δx Качественные Брак % качественных Отклонение от 50%
0.1 11 37 23% 27%
1.0 25 23 52% 2%
5.0 36 12 75% 25%

Оптимальный Δx = 1.0 (отклонение всего 2% от 50%)

8. Полный алгоритм работы СППР (исправленный)

// ШАГ 1: Базовый вердикт (вычисляется один раз, не зависит от текущего Δx)

base_delta = 1.0

Для каждой характеристики:

n = calculate_gradations(x, xmin, xmax, base_delta)

base_sum_log2 += log2(n)

base_Go = base_sum_log2 / N

base_P = exp(-ln2 / base_Go²)

is_quality = base_P ≤ 0.5 // ЭТО ОКОНЧАТЕЛЬНЫЙ ВЕРДИКТ

// ШАГ 2: Текущий расчет для визуализации (с заданным Δx)

current_delta = значение из слайдера

Для каждой характеристики:

n = calculate_gradations(x, xmin, xmax, current_delta)

current_sum_log2 += log2(n)

current_Co = current_sum_log2

current_Go = current_Co / N

current_P = exp(-ln2 / current_Go²)

// ШАГ 3: Отображение

Показываем пользователю: текущие n, log2(n), Co, Go, P

НО вердикт берем из ШАГА 1 (base_P ≤ 0.5)

✨ Преимущества такого подхода:

  • Пользователь может играться с Δx и видеть, как меняется чувствительность системы
  • Но при этом базовая классификация остается стабильной и соответствует методичке
  • Можно обучать систему, подбирая оптимальный Δx для наилучшего разделения

❓ Часто задаваемые вопросы

Вопрос: Почему при изменении Δx не меняется количество качественных товаров?

Ответ: Потому что вердикт вычисляется один раз с фиксированным Δx=1.0. Δx влияет только на отображение градаций, но не на сам факт качества. Это сделано в соответствии с методичкой.

Вопрос: Почему у некоторых товаров все градации = 2 даже при малом Δx?

Ответ: Это значит, что все характеристики товара находятся строго в пределах нормы. Для таких товаров даже при самом чувствительном Δx градации остаются равными 2.

Вопрос: Зачем тогда нужен слайдер Δx, если он не меняет вердикт?

Ответ: Слайдер показывает, насколько сильно система "чувствует" отклонения. При малом Δx даже небольшое отклонение дает большую градацию, при большом Δx - маленькую. Это помогает понять, какие характеристики вносят наибольший вклад и какова чувствительность системы.

Вопрос: Как обучить систему?

Ответ: Кнопка "Подобрать оптимальный Δx" находит такое значение Δx, при котором процент качественных товаров ближе всего к 50%. Это значение можно использовать как наиболее сбалансированное для данной выборки.
